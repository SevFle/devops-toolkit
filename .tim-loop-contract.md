# Implementation Contract: Fix 30 Confirmed Bugs

## Architecture Overview

The devops-toolkit repository has three independent code domains: (1) a Python orchestrator under `orchestrator/` that drives Claude CLI to implement spec-driven changes, with pytest tests under `orchestrator/tests/`; (2) GitHub Actions reusable workflows under `.github/workflows/` that handle CI, deploy, release, rollback, and auto-heal; (3) composite actions under `actions/` and shell scripts under `init/` that provide shared building blocks.

All Python code uses `dataclass(frozen=True)` for immutable data, `subprocess.run` for CLI invocation with a `_run_git` helper for git commands, and `StructuredLogger` for structured logging. Tests use pytest with `unittest.mock` (patch/MagicMock) and real temp git repos via `tempfile.TemporaryDirectory`. Workflows use `workflow_call` triggers, `appleboy/ssh-action` for remote deploys, and `fromJSON(format(...))` for runner labels.

## Shared Contracts

None required. All 30 bugs are isolated fixes within their respective files. No new shared types, interfaces, or constants are needed across partitions.

## Conventions

- **Naming:** Snake_case for Python functions/variables, kebab-case for workflow/action files. Test classes use `TestCamelCase`, test methods use `test_snake_case`.
- **Error handling:** Python uses `GitError` exception for git ops, `SystemExit` for fatal orchestrator errors. Subprocess calls use `check=True` or explicit returncode checks.
- **Test pattern:** Files named `test_<module>.py` in `orchestrator/tests/`. Fixtures via `@pytest.fixture`. Subprocess mocked with `@patch("module.subprocess.run")` or `@patch.object`. Real git repos created with `tempfile.TemporaryDirectory`.
- **Import style:** Relative imports within `lib/` package (e.g., `from lib.config import Config`). `pythonpath = ["."]` in pyproject.toml.
- **Config:** Frozen dataclass `Config` with `from_env()` classmethod, `validate()` returns `list[str]` of errors.
